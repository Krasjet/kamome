kamome

For document syntax related guides, see karasu.

kamome (カモメ) is the front-end componenti.e. the editor and CSS styles of Karasu. The editor is currently very alpha and need some polishment.

The source code can be found on GitHub. It’s mostly written in JavaScript and CSSI hate JavaScript, but I don’t like bringing in the interpreter of another language, either and assembled using WebPack.

1 tl;dr

Remember to click the Save button before you close the tab, and don’t click on the vim checkbox if you don’t know what it is.

2 Quick Tutorial (WIP)

Seriously, there is only three buttons you can press.

  1. Don’t use the editor on your phoneThough surprisingly, the editor seems to work on my iPad. Don’t use Internet Explorer.
  2. The left pane is the editor of the document and the right pane is the preview
  3. Click the render button to preview the document. Alternatively, you can press Ctrl-S while in the editor to preview itI added this shortcut only because I was pressing Ctrl-S too often. Access code is needed since this is hosted on my private server and rendering documents can be resource-consuming.
  4. [Important] Click the save button to save the document to server. Progress won’t be saved until you press this buttonThe browser will warn you if you try to close the tab with unsaved changes. Again, access code is needed. Every time you press the save button, the previous version will be backed up on the server, but only I can retreive it.
  5. If you somehow bypassed the browser’s warning and lost your progress. Go back to the editor page and press F12 to open the developer console. Go to the console page and run
sessionStorage['docBackup']

If this doesn’t help, I don’t have other ways to retreive the document.

  1. Both render and save will only be triggered when the document has changed.
  2. The rendering of is cached, so don’t be afraid to click render and save very often. The rendering is even faster than compiling native documents.
  3. Only click on the vim checkbox if you know how to use the vim editor. In theory, this option is just for myself. If you do use it, note that the clipboard has some problems, but there is no way to fix it due to browser restrictions.
  4. The grey text on the right of vim is the current status.
  5. If you change the edit in the URL into view. You can open the preview in full-screen mode.
